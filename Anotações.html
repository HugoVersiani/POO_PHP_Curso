INTRODUÇÃO À ORIENTAÇÃO À OBJETOS Paradigmas de programação são padrões, formas ou condutas de se programar. 1. Tipos de paradigmas: 1.1 Procedural/Estruturado/Imperativo: -Lógico; -Funcional; 1.2 Orientado à Objetos; -Concorrente; -Reativo; Cada línguagem possui suas proprias peculiaridades, o que às torna mais ou menos qualificadas para determinada finalidade. Dentre essas particularidades, está o suporte para os paradigmas de programação. PHP assim como JS aceita tanto para- -digma procedural quanto orientado a objetos. A linguagem dde programação JAVA, é fortemente tipada e nao aceita programação procedural 2. Classes e Objetos: 2.1 Classes são "formas" para criação de novos tipos de dados/variáveis: assim como existem strings, floats, ints, boolens, uma classe será um novo tipo de dado, que neces- sáriamente atenderá padrões nela definida. 2.2 Objetos são uma instância de uma classe, ou seja, um "bolo" feito na "forma", que é a classe. Quando se cria uma variável a partir de uma classe, se cria um objeto, e a classe é apenas o tipo desse objeto. 3. Orientação à Objetos: Possui pilares/princípios: 3.1 Pilar de Abstração: Abstração é a forma como se interpreta coisas do mundo real e transforma a ideia em código. Para se compreender melhor a abstração, é necessário estar por dentro de alguns conceitos: 3.1.1 - ENTIDADE: compreensão de que um objeto do mundo real precisa ser re- produzido para dentro da aplicação; 3.1.2 - IDENTIDADE: é a base de um objeto abstraido, que pode servir para cri- ação de outras instancia do objetos. 3.1.3 - CARACTERISTICAS: são os atributos do objeto, como categoria, titulo, valor, descrição, genero etc. 3.1.4 - AÇÕES: mais conhecidas como métodos, as ações definem o comportamento de um objeto, como mudar um valor, exibir uma imagem etc. 4. Getters e Setters: 4.1 Getters são funções que retornam um valor setado (ou não); 4.2 Setter são funções do tipo void, opu seja, não retornam nada porem executam uma instrução/atributo;
